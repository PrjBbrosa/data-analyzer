#### B2. Wrapper-QFrame WA_StyledBackground / paired-QSS-rule duplication

Files:
- `pyqt-ui/2026-04-26-action-button-on-group-title-needs-qframe-header.md`
- `pyqt-ui/2026-04-26-inspector-content-max-width-and-tinted-card-bleed.md` (rule 2)

Both lessons spell out, with similar code-fragment QSS, that:
- `setAttribute(Qt.WA_StyledBackground, True)` without a paired
  `QFrame#objectName { background: transparent; ... }` rule produces
  a white bleed against tinted contextual cards.
- A global `QFrame { background-color: #ffffff }` rule auto-polishes
  WA_StyledBackground on, so even removing the explicit attribute
  is not sufficient.

The action-button lesson is the originator; the
inspector-content-max-width lesson restates it as part of a
three-rule package (the rework-summary nature of that file makes
restatement natural).

- **Recommended action:** in inspector-content-max-width's "rule 2",
  shorten to "wrapper QFrames need paired QSS — see
  action-button-on-group-title-needs-qframe-header for the WA_Styled
  Background mechanics" and drop the duplicated body. Keep the
  cross-reference. This is purely cosmetic; both lessons remain
  active.
- **Risk:** very low.
